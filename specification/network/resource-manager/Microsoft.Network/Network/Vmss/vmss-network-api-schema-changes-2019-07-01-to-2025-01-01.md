# VMSS Network Interface & Public IP API — Schema Change Report (2019-08-01 → 2025-01-01)

## Scope and methodology

This report tracks substantial schema changes that impact the `vmssNetworkInterface.json` and `vmssPublicIpAddress.json` network APIs across the stable API versions from **2019-08-01** through **2025-01-01**.

- The two entry documents were parsed, and every schema reachable from their operation responses/parameters was resolved **recursively** through `$ref` (including cross-file references such as `network.json`, `networkInterface.json`, `publicIpAddress.json`, `loadBalancer.json`, `virtualNetwork.json`, etc.). A change in a schema referenced by a referenced schema (to any depth) is included.

- Consecutive versions in the range were compared. A change is reported as **substantial** when a property was **added** or **removed**, a property's **type changed**, or a property's **encoding changed** (e.g. `format` added/changed, a free-form `string` replaced by a modelled enum, or enum value set / `modelAsString` changed). Documentation-only, example-only and description-only edits are excluded.

- **Reference path** shows the first (shortest) reference chain from a top-level entry schema down to the changed schema, in `Schema.property.Schema.property…` form (a `[]` suffix marks an array-valued property). Only the first such path is shown even when a schema is reachable by several routes.

- **Changed @ / Previous @** link directly to the **line range** of the change inside the file on the `main` branch of `Azure/azure-rest-api-specs` (the changed property where it exists in that version, otherwise the enclosing schema block). The local working tree was verified identical to the current `origin/main` tip for this directory.
